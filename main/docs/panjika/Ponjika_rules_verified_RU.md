# Проверенные правила панчанги: Экадаши, Махадвадаши и переносы праздников

## Алгоритмическая спецификация по оригиналу «Шри Навадвипа-панджики», 540 Гаурабда

**Статус документа:** повторно сверено непосредственно с бенгальским сканом исходного PDF.  
**Основные страницы источника:** печатные страницы 4–9, соответствующие PDF-сканам 3–5.  
**Назначение:** спецификация для программной реализации календаря.

## 1. Что именно подтверждает исходный PDF

Исходный файл называется **«Шри Навадвипа-панджика — 540 Шри Гаурабда»**. В нём правила изложены как решения, основанные на «Шри Хари-бхакти-виласе», комментариях Шрилы Санатаны Госвами, трудах Шрилы Баладевы Видьябхушаны и пояснениях Шрилы Бхактисиддханты Сарасвати Тхакура.

Это важное ограничение: PDF не является технической документацией современной организации **Sri Chaitanya Saraswat Math**. Он подтверждает правила линии «Шри Навадвипа-панджики» и гаудия-вайшнавской традиции Шрилы Бхактисиддханты. Использовать их как основу календаря Сарасват Матха обоснованно, но окончательное тождество всех современных внутренних правил Сарасват Матха следует проверять по его фактическим календарям.

В этом документе используются три уровня достоверности:

- **[ПРЯМО]** — правило непосредственно напечатано в исходном PDF;
- **[ФОРМАЛИЗАЦИЯ]** — машинный предикат, непосредственно выражающий напечатанное правило;
- **[НЕ ОПРЕДЕЛЕНО]** — точная формула отсутствует в PDF, поэтому она не должна додумываться.

## 3. Минимальные астрономические данные

Движок должен предварительно вычислять границы титхи и накшатр независимо от календарных правил.

```ts
interface AstronomyProvider {
  sunrise(date: LocalDate, location: GeoLocation): Instant;
  sunset(date: LocalDate, location: GeoLocation): Instant;

  tithiAt(time: Instant): Tithi;
  nakshatraAt(time: Instant): Nakshatra;

  tithiOccurrenceAt(time: Instant): Interval;
  nakshatraOccurrenceAt(time: Instant): Interval;
}
```

Для правил нужны как минимум:

```ts
tithiAt(sunrise(D))
tithiAt(arunodaya(D))
tithiAt(sunrise(D + 1))
nakshatraAt(sunrise(D))
nakshatraAt(localMidnight(D))
start/end текущей титхи
start/end текущей накшатры
sunset(D)
```

## 4. Единицы времени и базовые определения

### 4.1. Арунодая

**[ПРЯМО, печатная стр. 5]**

```ts
arunodaya(D) = sunrise(D) - 96 minutes
```

Соотношение, указанное в тексте:

```text
2 мухурты = 4 данды = 96 минут
1 данда = 24 минуты
1 мухурта = 48 минут
```

### 4.2. Солнечный день

Для формализации правил удобно определить:

```ts
solarDay(D) = [sunrise(D), sunrise(D + 1))
```

### 4.3. Полная обычная титхи

**[ПРЯМО, печатная стр. 5]** обычная титхи, кроме Экадаши, называется полной, когда охватывает период от одного восхода до следующего — 60 данд.

**[ФОРМАЛИЗАЦИЯ]**

```ts
function isCompleteGeneralTithi(D: LocalDate, t: Tithi): boolean {
  return tithiAt(sunrise(D)) === t
      && tithiAt(sunrise(D + 1)) === t;
}
```

### 4.4. Чистая Экадаши

**[ПРЯМО, печатные стр. 5–6]** Экадаши оценивается не только на восходе, а в Арунодаю.

```ts
function isPureEkadashi(D: LocalDate): boolean {
  return tithiAt(arunodaya(D)) === EKADASHI;
}
```

Если в Арунодаю ещё Дашами, день отвергается, даже когда на восходе уже Экадаши.

### 4.5. Вриддха и кшая

**Вриддха-титхи** — титхи, присутствующая на двух последовательных восходах.

```ts
function isVriddhi(D: LocalDate, t: Tithi): boolean {
  return tithiAt(sunrise(D)) === t
      && tithiAt(sunrise(D + 1)) === t;
}
```

**Кшая-титхи** — титхи, полностью прошедшая между двумя восходами и не присутствующая ни на одном из них.

```ts
function isKshayaBetweenSunrises(D: LocalDate, t: Tithi): boolean {
  const x = occurrenceOf(t, solarDay(D));
  return x != null
      && x.start > sunrise(D)
      && x.end < sunrise(D + 1);
}
```

## 5. Виддха: общее правило

### 5.1. Виды виддхи

**[ПРЯМО, печатные стр. 4–5]**

- **пурва-виддха** — примесь предыдущей титхи;
- **пара-виддха** — примесь последующей титхи.

В тексте также упомянуты сурьодая-виддха и полуночная виддха, однако критерий полуночной виддхи отвергается как необоснованный для данной системы.

### 5.2. Главное разделение

**Для Экадаши:** отвергается Дашами, присутствующая в Арунодаю.

**Для Джанмаштами и остальных врат:** отвергается предыдущая титхи, присутствующая на восходе.

```ts
function isValidEkadashiDay(D: LocalDate): boolean {
  return tithiAt(arunodaya(D)) === EKADASHI;
}

function isValidOrdinaryFestivalDay(D: LocalDate, target: Tithi): boolean {
  return tithiAt(sunrise(D)) === target;
}
```

Критически важно не применять арунодая-правило ко всем праздникам.

## 6. Модель результата

Дата праздника, дата поста, время поклонения и дата параны могут различаться. Их нельзя хранить одним полем.

```ts
interface ObservanceResult {
  festivalDate: LocalDate;
  fastDate?: LocalDate;
  worshipWindow?: Interval;
  paranaDate?: LocalDate;
  paranaNotBefore?: Instant;
  paranaNotAfter?: Instant;

  classification: RuleCode;
  sourcePages: number[];
  confidence: "EXPLICIT" | "FORMALIZED" | "UNRESOLVED";
  explanation: string;
}
```

## 7. Базовый алгоритм Экадаши

### 7.1. Проверка чистоты

```ts
function classifyEkadashiSunriseDay(D: LocalDate):
  | "PURE"
  | "DASHAMI_AT_ARUNODAYA"
  | "NO_EKADASHI_AT_SUNRISE" {

  if (tithiAt(sunrise(D)) !== EKADASHI) {
    return "NO_EKADASHI_AT_SUNRISE";
  }

  if (tithiAt(arunodaya(D)) === EKADASHI) {
    return "PURE";
  }

  if (tithiAt(arunodaya(D)) === DASHAMI) {
    return "DASHAMI_AT_ARUNODAYA";
  }

  throw new UnresolvedCalendarCase();
}
```

### 7.2. Перенос при Дашами в Арунодаю

**[ПРЯМО]** день с Дашами в Арунодаю отвергается.  
**[ФОРМАЛИЗАЦИЯ]** пост переносится на следующий допустимый день цикла Экадаши–Двадаши.

```ts
function resolveBasicEkadashi(window: LocalDate[]): LocalDate {
  for (const D of window) {
    if (tithiAt(arunodaya(D)) === EKADASHI) {
      return D;
    }
  }

  const contaminated = window.find(D =>
    tithiAt(sunrise(D)) === EKADASHI &&
    tithiAt(arunodaya(D)) === DASHAMI
  );

  if (contaminated) {
    return contaminated.plusDays(1); // перенос на следующий день
  }

  throw new UnresolvedCalendarCase(
    "PDF не формулирует отдельное правило для кшая-Экадаши"
  );
}
```

Перед окончательным выбором необходимо проверить специальные Махадвадаши.

## 8. Харивасара и парана Экадаши

### 8.1. Первая четверть Двадаши

**[ПРЯМО, печатная стр. 5]** первая четверть Двадаши входит в период поста Экадаши и также называется Харивасарой.

```ts
function hariVasaraEnd(dvadashi: Interval): Instant {
  return dvadashi.start
    + (dvadashi.end - dvadashi.start) / 4;
}
```

Следовательно:

```ts
paranaNotBefore >= hariVasaraEnd(dvadashi)
```

### 8.2. Утренний восход

Календарные записи PDF назначают парану на следующее утро, однако в пояснительных страницах нет единой напечатанной формулы вида:

```ts
max(nextSunrise, hariVasaraEnd)
```

Для программной реализации допустимо использовать:

```ts
paranaNotBefore = max(
  sunrise(paranaDate),
  hariVasaraEnd(dvadashi)
);
```

но статус этого выражения — **[ФОРМАЛИЗАЦИЯ]**, а не буквальная цитата из PDF.

### 8.3. Верхняя граница

**[НЕ ОПРЕДЕЛЕНО]** исходный раздел правил не устанавливает универсальную верхнюю границу параны через 1/3 или 1/5 пратах-калы. В самом календаре интервалы приводятся для конкретных дат, но общей формулы в проверенных страницах нет.

## 9. Восемь Махадвадаши

**[ПРЯМО, печатная стр. 9]** перечислено восемь Махадвадаши.

Четыре титхи-зависимые:

1. Унмилани;
2. Банджули / Вьянджули (бенг. `বঞ্জুলী`);
3. Трисприша;
4. Пакшавардхини.

Четыре накшатра-зависимые:

1. Джая;
2. Виджая;
3. Джаянти;
4. Папанашини.

### 9.1. Унмилани

**[ПРЯМО]** полная Экадаши возрастает и касается утра следующего дня.

В конце той же страницы полная Экадаши в данном контексте описана как 64 данды — от предыдущей Арунодаи до следующего восхода.

**[ФОРМАЛИЗАЦИЯ]**

```ts
function isUnmilani(E: LocalDate): boolean {
  return tithiAt(arunodaya(E)) === EKADASHI
      && tithiAt(sunrise(E + 1)) === EKADASHI;
}
```

```ts
fastDate = E + 1;
classification = "UNMILANI_MAHADVADASHI";
```

### 9.2. Банджули / Вьянджули

**[ПРЯМО]** Экадаши должна быть полной; Двадаши также должна быть полной и возрастать в сторону Трайодаши.

**[ФОРМАЛИЗАЦИЯ]** обычное машинное выражение «полной Двадаши»:

```ts
function isVyanjuli(E: LocalDate): boolean {
  const D = E + 1;

  return isPureEkadashi(E)
      && tithiAt(sunrise(D)) === DVADASHI
      && tithiAt(sunrise(D + 1)) === DVADASHI;
}
```

```ts
fastDate = E + 1; // день Двадаши
classification = "VYANJULI_MAHADVADASHI";
```

Название в бенгальском оригинале напечатано как `বঞ্জুলী`; русскую форму следует хранить отдельно от внутреннего кода.

### 9.3. Трисприша

**[ПРЯМО]** в одном дне соприкасаются Экадаши, Двадаши и в конце ночи Трайодаши.

```ts
function isTrisprisha(D: LocalDate): boolean {
  const sr0 = sunrise(D);
  const sr1 = sunrise(D + 1);

  return tithiAt(sr0) === EKADASHI
      && containsTithiBetween(DVADASHI, sr0, sr1)
      && tithiAt(sr1) === TRAYODASHI;
}
```

```ts
fastDate = D;
paranaDate = D + 1; // Трайодаши
classification = "TRISPRISHA_MAHADVADASHI";
```

**[НЕ ОПРЕДЕЛЕНО]** выражение «в конце ночи» не снабжено числовой границей. PDF не указывает, считать ли «концом ночи» конкретную мухурту, Арунодаю или весь интервал до восхода.

### 9.4. Пакшавардхини

**[ПРЯМО]** если Амавасья или Пурнима возрастает, Двадаши на следующий день после предшествующей чистой Экадаши называется Пакшавардхини.

```ts
function terminalTithiIsVriddhi(
  P: LocalDate,
  t: typeof AMAVASYA | typeof PURNIMA
): boolean {
  return tithiAt(sunrise(P)) === t
      && tithiAt(sunrise(P + 1)) === t;
}
```

```ts
if (terminalTithiIsVriddhi(P, AMAVASYA_OR_PURNIMA)) {
  const E = immediatelyPrecedingPureEkadashi(P);
  fastDate = E + 1; // Двадаши
  classification = "PAKSHAVARDHINI_MAHADVADASHI";
}
```

### 9.5. Четыре накшатра-Махадвадаши

Они возникают только на **Шукла-Двадаши**:

| Название | Накшатра |
|---|---|
| Джая | Пунарвасу |
| Виджая | Шравана |
| Джаянти | Рохини |
| Папанашини | Пушья |

#### Условие накшатры

Оригинал различает два случая:

1. если соответствующая накшатра вступает в действие от восхода Шукла-Двадаши, дальнейшая продолжительность несущественна;
2. если она началась до восхода, она должна продолжаться весь день и ночь Двадаши — 60 данд — либо дольше.

Точная программная трактовка выражения «вступает в действие от восхода» должна сохраняться как отдельный режим, а не заменяться произвольным условием «накшатра встречалась когда-либо в течение дня».

#### Дополнительное условие продолжительности Двадаши

**[ПРЯМО]**

```ts
function dvadashiDurationQualifies(
  D: LocalDate,
  n: Nakshatra
): boolean {

  if (n === PUNARVASU || n === ROHINI || n === PUSHYA) {
    return dvadashiEnd(D) >= sunset(D);
  }

  if (n === SHRAVANA) {
    return dvadashiEnd(D) >= oneAndHalfPraharAfterDayStart(D);
  }

  return false;
}
```

**[НЕ ОПРЕДЕЛЕНО]** PDF не переводит полтора прахара в современные часы. Возможная реализация `восход + 4:30` требует отдельного подтверждения принятой системой единиц и не должна быть скрытым предположением.

## 10. Шри Джанмаштами

### 10.1. Базовая чистота

**[ПРЯМО, печатные стр. 5–7]** Аштами отвергается, если на восходе ещё Саптами. Наличие Саптами в Арунодаю само по себе не делает Аштами недействительной.

```ts
function isValidJanmashtamiDay(D: LocalDate): boolean {
  return tithiAt(sunrise(D)) === KRISHNA_ASHTAMI;
}
```

Не требуется универсально ждать Аштами, соединённую с Навами. Чистая Аштами на восходе уже допустима.

### 10.2. Возрастающая Аштами

Проверенное правило выбора:

1. если чистая Аштами возрастает и на следующий день соединяется с Рохини, выбирается следующий день, даже если предыдущий день — понедельник или среда;
2. если оба дня имеют чистую Аштами и Рохини, выбирается день, в полночь которого присутствует Рохини;
3. если Рохини присутствует в полночь обоих дней, выбирается предыдущий день.

```ts
function resolveVriddhiJanmashtami(A: LocalDate, B: LocalDate): LocalDate {
  assert(B === A + 1);
  assert(tithiAt(sunrise(A)) === KRISHNA_ASHTAMI);
  assert(tithiAt(sunrise(B)) === KRISHNA_ASHTAMI);

  const rA = overlapsSolarDay(A, ROHINI);
  const rB = overlapsSolarDay(B, ROHINI);

  if (!rA && rB) return B;
  if (rA && !rB) return A; // естественная симметричная формализация

  if (rA && rB) {
    const mA = nakshatraAt(localMidnight(A)) === ROHINI;
    const mB = nakshatraAt(localMidnight(B)) === ROHINI;

    if (mA && !mB) return A;
    if (!mA && mB) return B;
    if (mA && mB) return A;
  }

  throw new UnresolvedCalendarCase(
    "Проверенный абзац не задаёт полный порядок для комбинации без Рохини"
  );
}
```

Строка `if (rA && !rB) return A` — логическая симметричная формализация. Оригинал отдельно акцентирует противоположный случай — преимущество второго дня при Рохини во второй день.

### 10.3. Парана Джанмаштами

**[ПРЯМО, печатная стр. 7]** парану нельзя совершать, пока продолжается Аштами или Рохини.

```ts
janmashtamiParanaNotBefore = max(
  ashtami.end,
  rohini.end
);
```

Некоторые вайшнавы совершают парану после завершения праздника явления Бхагавана. Универсальная верхняя граница в этом разделе не дана.

## 11. Шри Раманавами

### 11.1. Обычное правило

```ts
normalRamanavami(D) =
  month(D) === CHAITRA &&
  paksha(D) === SHUKLA &&
  tithiAt(sunrise(D)) === NAVAMI;
```

### 11.2. Исключение при кшая Дашами

**[ПРЯМО, печатная стр. 8]** если из-за кшая-титхи после дня Навами на следующий день уже приходится пост Шукла-Экадаши, Раманавами соблюдается в Навами, загрязнённую Аштами.

**[ФОРМАЛИЗАЦИЯ]**

```ts
function isRamanavamiKshayaException(D: LocalDate): boolean {
  const sr0 = sunrise(D);
  const sr1 = sunrise(D + 1);

  return tithiAt(sr0) === SHUKLA_ASHTAMI
      && containsTithiBetween(SHUKLA_NAVAMI, sr0, sr1)
      && containsTithiBetween(SHUKLA_DASHAMI, sr0, sr1)
      && tithiAt(sr1) === SHUKLA_EKADASHI;
}
```

```ts
if (isRamanavamiKshayaException(D)) {
  festivalDate = D;
  fastDate = D;
}
```

Это специальное исключение. Его нельзя превращать в общее правило для всех праздников.

## 12. Шри Вамана-Двадаши

**[ПРЯМО, печатные стр. 4 и 8]** титхи явления Шри Ваманадевы — Бхадра Шукла-Двадаши.

### 12.1. Без Махадвадаши

```ts
fastDate = ekadashiDate;
worshipWindow = interval(
  ekadashiNightStart,
  firstHalfOfDvadashiEnd
);
paranaDate = dvadashiDate;
paranaNotBefore = endOfVamanaWorship;
```

То есть:

- пост начинается в Экадаши;
- поклонение Ваманадеве совершается ночью Экадаши либо в первой половине Двадаши;
- парана проводится в Двадаши после поклонения.

### 12.2. Если Двадаши является Махадвадаши

Общее правило текста: при возникновении Махадвадаши Экадаши оставляют и постятся в Двадаши.

```ts
if (isMahadvadashi(dvadashiDate)) {
  fastDate = dvadashiDate;
  festivalDate = dvadashiDate;
  paranaDate = dvadashiDate + 1;
}
```

## 13. Шри Говардхана-пуджа

### 13.1. Основной день

**[ПРЯМО, печатная стр. 8]** Говардхана-пуджа совершается в первой половине дня, утром которого присутствует Картика Шукла-Пратипада.

```ts
function normalGovardhanaDay(D: LocalDate): boolean {
  return month(D) === KARTIKA
      && tithiAt(sunrise(D)) === SHUKLA_PRATIPADA;
}
```

### 13.2. Вриддха Пратипада

Если Пратипада присутствует на двух восходах, выбирается второй день — Пратипада, соединённая с Двитией.

```ts
if (isVriddhi(D, SHUKLA_PRATIPADA)) {
  festivalDate = D + 1;
}
```

### 13.3. Кшая Пратипада

Если Пратипада полностью проходит между восходами, текст разрешает провести Говардхана-пуджу на Двитии вследствие кшая-титхи.

```ts
if (isKshayaBetweenSunrises(D, SHUKLA_PRATIPADA)
    && tithiAt(sunrise(D + 1)) === SHUKLA_DVITIYA) {
  festivalDate = D + 1;
}
```

### 13.4. Запрет Амавасья-юкта Пратипады

Собственно Говардхана-пуджа не проводится на Пратипаде, соединённой с Амавасьей — на пурва-виддха Пратипаде.

```ts
if (tithiAt(sunrise(D)) === AMAVASYA) {
  rejectAsGovardhanaPujaDay(D);
}
```

### 13.5. Не смешивать с го-пуджей и играми с коровами

PDF отдельно обсуждает го-пуджу и го-криду. При определённой комбинации они могут выполняться в предыдущий день, а Говардхана-пуджа — на следующее утро. Это должны быть разные события в модели данных.

```ts
EventCode.GO_PUJA
EventCode.GO_KRIDA
EventCode.GOVARDHANA_PUJA
```

Точная астрономическая формула «возможности видеть Луну» в PDF не определена.

## 14. Явления Гауры, Нитьянанды и Баладевы

**[ПРЯМО, печатная стр. 4]** гаудия-вайшнавы соблюдают пост в титхи явления:

- Шри Гаурасундары;
- Шри Баладевы Прабху;
- Шри Нитьянанды Прабху.

Для чистоты применяется общее правило не-Экадаши: целевая титхи должна быть на восходе; арунодая-виддха к этим вратам не применяется.

Однако проверенные страницы не дают полного алгоритма выбора первого или второго дня для всех возможных вриддха-комбинаций этих праздников и не устанавливают универсальную формулу их параны.

## 15. Рекомендуемый порядок правил в движке

```text
1. Рассчитать точные границы титхи и накшатр.
2. Рассчитать локальные восходы, закаты, полуночи и Арунодаю.
3. Определить вриддха и кшая по отношению к восходам.
4. Для конкретного праздника сначала применить его специальное правило:
   - Джанмаштами;
   - Раманавами;
   - Вамана-Двадаши;
   - Говардхана-пуджа.
5. Для цикла Экадаши проверить специальные Махадвадаши.
6. Если специальной Махадвадаши нет, применить правило Дашами в Арунодаю.
7. Для остальных врат применить правило предыдущей титхи на восходе.
8. Отдельно назначить festivalDate, fastDate, worshipWindow и paranaDate.
9. Не разрешать парану внутри Харивасары.
10. Не вычислять верхнюю границу параны по формуле, которой нет в источнике.
```

## 16. Сводный псевдокод

```ts
function resolveObservance(
  event: EventDefinition,
  window: LocalDate[],
  astro: AstronomyProvider
): ObservanceResult {

  switch (event.code) {
    case "JANMASHTAMI":
      return resolveJanmashtami(window, astro);

    case "RAMANAVAMI":
      return resolveRamanavami(window, astro);

    case "VAMANA_DVADASHI":
      return resolveVamanaDvadashi(window, astro);

    case "GOVARDHANA_PUJA":
      return resolveGovardhanaPuja(window, astro);

    case "EKADASHI": {
      const special = detectMahadvadashi(window, astro);
      if (special) return special;
      return resolveOrdinaryEkadashi(window, astro);
    }

    default:
      return resolveSunriseTithiFestival(event, window, astro);
  }
}
```

```ts
function resolveSunriseTithiFestival(
  event: EventDefinition,
  window: LocalDate[],
  astro: AstronomyProvider
): ObservanceResult {

  const dates = window.filter(D =>
    astro.tithiAt(astro.sunrise(D)) === event.targetTithi
  );

  if (dates.length === 1) {
    return resultFor(dates[0], "SUNRISE_TITHI_RULE");
  }

  if (dates.length === 2) {
    throw new EventSpecificRuleRequired("VRIDDHI");
  }

  if (dates.length === 0) {
    throw new EventSpecificRuleRequired("KSHAYA");
  }

  throw new InvalidAstronomyData();
}
```

## 17. Что нельзя считать установленным данным PDF

Следующие параметры не следует зашивать в код как «правила Сарасват Матха» без дополнительного источника:

1. универсальная верхняя граница параны через 1/3 или 1/5 пратах-калы;
2. точная числовая граница выражения «конец ночи» для Трисприша;
3. современное часовое значение полутора прахар для Виджая-Махадвадаши;
4. универсальный выбор первого или второго дня для любого вриддха-праздника;
5. универсальный перенос любого кшая-праздника;
6. правила для мест без нормального восхода Солнца;
7. доказательство полного совпадения с современным внутренним алгоритмом Sri Chaitanya Saraswat Math;
8. точная формула лунной видимости для раздельных го-пуджи и го-криды;
9. полный порядок при редких комбинациях Джанмаштами, которые не разобраны в проверенном абзаце.

## 19. Краткая итоговая спецификация

```text
ЭКАДАШИ:
- Арунодая = восход - 96 минут.
- Дашами в Арунодаю загрязняет и отменяет этот день поста.
- Благоприятная накшатра не очищает такую Экадаши.
- Первая четверть Двадаши входит в Харивасару.

ОСТАЛЬНЫЕ ВРАТЫ:
- Проверяется предыдущая титхи на восходе.
- Предыдущая титхи в Арунодаю сама по себе не загрязняет праздник.
- Полуночная виддха не принимается.

ДЖАНМАШТАМИ:
- Нужна Аштами на восходе.
- При двух днях учитывается Рохини и её присутствие в полночь.
- Если Рохини в полночь обоих дней, выбирается первый день.
- Парана только после окончания Аштами и Рохини.

РАМАНАВАМИ:
- Обычно Навами на восходе.
- При кшая Дашами и Экадаши на следующем восходе допускается Аштами-виддха Навами.

ВАМАНА-ДВАДАШИ:
- Без Махадвадаши пост в Экадаши.
- Поклонение ночью Экадаши или в первой половине Двадаши.
- Парана в Двадаши после поклонения.

ГОВАРДХАНА-ПУДЖА:
- Картика Шукла-Пратипада утром.
- При вриддха выбирается второй день, соединённый с Двитией.
- При кшая допускается Двития.
- Нельзя проводить на Амавасья-юкта Пратипаде.

МАХАДВАДАШИ:
- Унмилани, Банджули/Вьянджули, Трисприша, Пакшавардхини.
- Джая, Виджая, Джаянти, Папанашини — только Шукла-Двадаши с указанными накшатрами.
```

## 20. Заключение о точности

После повторной сверки с оригинальным PDF основа предыдущей спецификации — различие между арунодая-виддхой Экадаши и сурьодая-виддхой остальных врат — подтверждена.

Исправлены или ограничены наиболее важные места:

- устранено ложное впечатление, будто универсальная формула параны полностью напечатана в PDF;
- уточнены условия накшатра-Махадвадаши;
- исправлена область применения Шукла-Двадаши;
- уточнено время поклонения Ваманадеве;
- отделены Говардхана-пуджа, го-пуджа и го-крида;
- исключены универсальные переносы, которых оригинал не устанавливает;
- сохранены явные зоны неопределённости вместо догадок.
